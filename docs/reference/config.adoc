= Configuration Reference

Place a `docserver.config.js` file in your docs directory to customise the server.
The file is optional — adocserver works without it.

[source,javascript]
----
// docserver.config.js
export default {
  title: 'My Project',
  accent: '#2563eb',
  accentStrong: '#1d4ed8',
  logo: '/logo.png',
}
----

== Options

=== `title`

[horizontal]
Type:: `string`
Default:: `'Docs'`

The site name. Shown as the sidebar brand when no logo is set, and appended to every page title (`Page Title — My Project`).

=== `accent`

[horizontal]
Type:: `string` (CSS colour)
Default:: `'#a81d2d'`

Primary accent colour. Used for active nav items, links, the sidebar resize handle, and admonition borders.

=== `accentStrong`

[horizontal]
Type:: `string` (CSS colour)
Default:: `'#7f1321'`

Hover/active variant of the accent colour.

=== `logo`

[horizontal]
Type:: `string` (URL)
Default:: _(none — shows title text)_

URL of the logo image shown in the sidebar brand area.
The easiest way to serve a local image is to place it in a `public/` subdirectory of your docs folder
(Vite serves `public/` as the root automatically):

[source]
----
docs/
├── index.adoc
├── docserver.config.js
└── public/
    └── logo.png        ← served as /logo.png
----

[source,javascript]
----
// docserver.config.js
export default {
  logo: '/logo.png',
}
----

=== `css`

[horizontal]
Type:: `string` (URL)
Default:: _(none — auto-detects `custom.css` in your docs directory)_

URL of a stylesheet injected after the built-in styles.
Place the file in `public/` so Vite serves it, then reference it by its root-relative URL:

[source,javascript]
----
export default {
  css: '/themes/corporate.css',
}
----

If a `custom.css` file exists in your docs directory it is picked up automatically without any config entry.
See xref:../guide/theming.adoc[Theming] for available CSS custom properties.

=== `template`

[horizontal]
Type:: `string` (path, relative to docs directory)
Default:: _(none — auto-detects `template.html` in your docs directory)_

Path to a custom HTML template file.
adocserver fills `{{double-brace}}` placeholders with page content:

[cols="1,3"]
|===
| Slot | Contents

| `{{title}}`   | Page title string
| `{{head}}`    | HMR client, MathJax, highlight.js stylesheet, favicon
| `{{sidebar}}` | Rendered sidebar HTML
| `{{body}}`    | Converted AsciiDoc HTML
| `{{scripts}}` | highlight.js init + sidebar resize script
|===

[source,javascript]
----
export default {
  template: 'themes/my-template.html',
}
----

If a `template.html` file exists in your docs directory it is picked up automatically.
See xref:../guide/theming.adoc[Theming] for a full walkthrough.

== Full example

[source]
----
docs/
├── index.adoc
├── docserver.config.js
├── public/
│   └── logo.svg
├── guide/
│   ├── index.adoc
│   └── ...
└── reference/
    ├── index.adoc
    └── ...
----

[source,javascript]
----
// docserver.config.js
export default {
  title: 'My Project',
  accent: '#0f766e',
  accentStrong: '#0d6561',
  logo: '/logo.svg',
}
----
